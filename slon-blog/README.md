# Блог для slon.life — двомовний

Двомовний блог-стрічка історій про Ігоря «Слона» Утюжа на Eleventy, інтегрований з твоєю існуючою системою `i18n.json` + `lang.js`.

---

## Як це працює (коротко)

- Кожна історія — **один Markdown-файл** з обома мовами одразу (поля `body_uk` і `body_en`).
- Eleventy при збірці генерує **одну HTML-сторінку для кожного поста**, всередині якої лежать обидві мови.
- На сторінці твій існуючий `lang.js` перемикає видимість блоків `data-lang-content="uk"` / `data-lang-content="en"` без перезавантаження.
- Інтерфейс блогу (меню, фільтри, кнопки) перекладається через `i18n.json` за стандартними `data-i18n` атрибутами — точно так само, як на решті сайту.
- URL поста — один: `/stories/slug.html`. Працює і для UA, і для EN.

## Що було додано в i18n.json

14 нових ключів у стилі твоїх `memories_*`:

```
menu_stories
stories_kicker
stories_title
stories_subtitle
stories_filter_all
stories_filter_childhood
stories_filter_maidan
stories_filter_service
stories_filter_tribute
stories_read_more
stories_empty
story_nav_prev
story_nav_next
story_back_to_list
```

Я додав їх безпосередньо до файлу `i18n.json` у твоєму проєкті. Решта ключів (`menu_about`, `footer_*`, тощо) — твої, я нічого не чіпав.

**Коли заливатимеш на хостинг — перезаливай оновлений `i18n.json`** (або акуратно зливай додані 14 ключів зі своєю версією, якщо ти її редагував між тим).

## Що було додано в lang.js

Один новий блок — підтримка `data-lang-content`:

```js
// Двомовний контент (для блогу)
document.querySelectorAll("[data-lang-content]").forEach(el => {
    const elLang = normalizeLang(el.getAttribute("data-lang-content"));
    el.style.display = (elLang === currentLang) ? "" : "none";
});
```

Усе інше в `lang.js` без змін. Новий блок ніяк не впливає на існуючі сторінки (`biography.html`, `index.html`) — на них просто немає елементів з атрибутом `data-lang-content`, і блок ніколи не спрацює.

## Як написати нову історію

Створи файл `_posts/YYYY-MM-DD-slug.md`:

```markdown
---
slug: kryms`ka-mriya
date: 2026-05-15
category: service
photo: /img/stories/crimea-map.jpg
photo_portrait: false

title:
  uk: Кримська мрія
  en: Crimean dream

author:
  uk: Євгенія, наречена
  en: Yevheniia, fiancée

lede:
  uk: Ігор готувався до подорожі, яка мала статися після перемоги.
  en: Ihor was preparing for a journey that was to happen after victory.

photo_caption:
  uk: Карта майбутнього маршруту
  en: Map of the future route

body_uk: |
  Тут починається текст історії українською. Звичайний абзац.

  Ще один абзац — між абзацами просто порожній рядок.

  > Ключова цитата (виділяється стилем pull-quote).

  І знову звичайний текст.

body_en: |
  Here begins the English version. A regular paragraph.

  Another paragraph — just an empty line between them.

  > Key quote (rendered as pull-quote).

  And regular text again.
---
```

**Важливо:** після `body_uk: |` і `body_en: |` усі рядки тексту повинні починатися з відступу (хоча б 2 пробіли). Це YAML-вимога, і саме так ми кажемо «це багаторядкове поле з форматуванням».

### Поля frontmatter

| Поле | Обов'язкове | Опис |
|---|---|---|
| `slug` | так | Короткий ідентифікатор (стає URL-ом: `/stories/SLUG.html`). Тільки латиниця, цифри, дефіси. |
| `date` | так | `YYYY-MM-DD` — впливає на сортування і шапку поста |
| `category` | так | Одне з: `childhood`, `maidan`, `service`, `tribute` |
| `title.uk` / `title.en` | так | Заголовок українською і англійською |
| `body_uk` / `body_en` | так | Тіло поста двома мовами (Markdown) |
| `author.uk` / `author.en` | ні | Хто розповідає |
| `lede.uk` / `lede.en` | ні | Анонс — показується у стрічці і в hero посту |
| `photo` | ні | Шлях до головного фото, наприклад `/img/stories/file.jpg` |
| `photo_caption.uk` / `photo_caption.en` | ні | Підпис під фото обома мовами |
| `photo_portrait` | ні | `true`, якщо фото вертикальне (без обрізання) |

### Markdown у тілі поста

- Звичайний абзац — просто текст з порожнім рядком навколо.
- Цитата (стилізована як pull-quote) — рядок, що починається з `>`.
- Жирний — `**текст**`. Курсив — `*текст*` (у блозі курсив автоматично стає золотим).
- Внутрішнє посилання — `[Біографія](/biography.html)`.

### ✨ Типографічні «голоси» — для виділення думок і нотаток

Окрім звичайної цитати (`>`) та курсиву (`*текст*`), у блозі доступні три додаткові стилі для особливих фрагментів. Синтаксис: пишеш абзац як зазвичай, у кінці додаєш `{.назва-класу}`.

**`{.thought}` — внутрішній голос, думка, фраза з листа Ігоря.**
Великий серифний курсив з тонкою золотою лінією зліва. Виглядає як вставка з книжки.

```markdown
Тату, на воді не можна збрехати. Або ти йдеш правильно, або тебе несе. {.thought}
```

**`{.handwritten}` — дослівний рукописний запис.**
Шрифт Caveat (наближений до рукопису, з підтримкою кирилиці), золотий, по центру. Найсильніший емоційний акцент — варто використовувати рідко: для записок з блокнота, останніх СМС, підписів на фото.

```markdown
Південний Буг. Усі разом. Як завжди. {.handwritten}
```

**`{.aside}` — авторська ремарка від оповідача.**
Дрібним приглушеним курсивом між пунктирними лініями. Для контексту: коли, де, за яких обставин це було сказано чи відбулося.

```markdown
Цей запис ми знайшли у блокноті, який передали побратими разом з його речами. {.aside}
```

**Важливо про синтаксис:**
- `{.клас}` ставиться **в кінці абзацу через пробіл**, у тому ж рядку, що останнє речення.
- Не плутай з Kramdown-синтаксисом `{:.клас}` (з двокрапкою) — у нашій конфігурації працює саме без двокрапки.
- На один абзац можна тільки один клас з цих трьох (вони взаємно виключні візуально).

Усі три голоси описані у `voices.css`. Якщо схочеш змінити стиль — редагуй цей файл.

## Що залити на хостинг

1. **Усе вміст `_site/`** заливай у корінь slon.life — поряд з `index.html`, `biography.html`, тощо.
2. **Перезалий оновлений `i18n.json`** і `lang.js` (вони в `_site/` — Eleventy їх скопіював).
3. **Додай посилання на блог у меню решти сторінок.** Відкрий `index.html`, `biography.html`, `memories.html` тощо і в блок `<nav class="header-nav">` додай після «Біографія»:

```html
<a href="stories.html" data-i18n="menu_stories">Історії</a>
```

Все. Ключ `menu_stories` уже є в `i18n.json`, тому переклад працюватиме на всіх сторінках одразу.

## Часті питання

**Як додати нову рубрику?**
1. У `i18n.json` додай два ключі: `stories_filter_NEWNAME` (UA + EN).
2. У `stories.njk` додай нову кнопку `<button data-filter="NEWNAME" data-i18n="stories_filter_NEWNAME">Назва</button>`.
3. Готово — у постах можеш писати `category: NEWNAME`.

**Як додати ще одну мову (DE, PL, …)?**
1. У `i18n.json` додай новий ключ верхнього рівня (`"de": {...}`) з усіма перекладами.
2. У `lang.js` додай мову до `SUPPORTED_LANGS = ["uk", "en", "de"]`.
3. У постах додавай поля `title.de`, `body_de` тощо. У шаблонах — обгортки `<span data-lang-content="de">{{ ... }}</span>`.
4. У хедерах і футерах додай кнопки `<button data-lang="de">DE</button>`.

**Чи можна додати коментарі?**
Так. У шаблоні `post.njk` після блоку навігації можеш додати `<div id="comments"></div>` і підключити Disqus, або повторити паттерн з `memories.html` через Supabase — там у тебе вже все налаштовано. Скажи, якщо треба — допоможу.

**Що буде з SEO для англійської версії?**
Оскільки в нас одна сторінка з двома мовами, пошуковикам видно UA-контент за замовчуванням (з тегами hreflang можна вказати, що це двомовний документ). Якщо потрібен жорсткий SEO для EN-версій (окремі URL для англійських постів) — це можна зробити, але це інша архітектура. Для меморіального сайту достатньо поточного варіанту.

**Чи зміниться щось у твоїх існуючих сторінках (`biography.html` тощо)?**
Ні. Тільки `i18n.json` і `lang.js` потрібно перезалити з оновленнями (нові ключі для блогу, новий блок для `data-lang-content`). Усі існуючі ключі і функції збережені без змін.
